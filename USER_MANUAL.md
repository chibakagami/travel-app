## 1. 安裝與啟動

### 線上開啟
直接從瀏覽器訪問 GitHub Pages 連結即可使用，所有資料儲存於瀏覽器本機（localStorage），不會上傳到任何伺服器。

### 加到主畫面（建議）
- **iPhone（Safari）：** 開啟頁面 → 點分享 → 「加入主畫面」
- **Android（Chrome）：** 開啟頁面 → 點 ⋯ → 「加入主畫面」

加入主畫面後會顯示為一個寶貝球圖示，點擊即可全螢幕啟動。

---

## 2. 首頁：寶貝球選單

啟動後會看到紅色背景的首頁，畫面中央有一顆會輕輕跳動的**寶貝球**。

### 操作方式
- **點擊寶貝球**：展開三個選項按鈕
- **點擊紅色背景**：收合選單
- **首頁右下角**：顯示版本號 V1.01

### 三個選項
| 按鈕 | 功能 |
|---|---|
| 🗺 **冒險開始** | 開始規劃新的旅行 |
| 🧭 **繼續冒險** | 回到目前進行中的旅行（無進行中時為灰色不可點） |
| 📸 **美好回憶** | 查看已封存的旅行紀錄 |

> 若已有進行中旅行又點「冒險開始」，會詢問是否將前次旅行封存到美好回憶。

## 3. 建立新旅行

點擊「冒險開始」後進入設定頁，需填寫：

| 欄位 | 說明 |
|---|---|
| **目的地** | 必填，例如「東京」「首爾」 |
| **出發日期 / 回程日期** | 必填 |
| **當地貨幣** | 用於記帳與匯率換算（JPY / KRW / THB / USD…等） |
| **旅遊預算** | 可留空；填寫後記帳頁會顯示用量百分比 |

按 **「開始冒險 ▶」** 進入主畫面，自動產生對應天數的 Day 分頁。

## 4. 準備 ✅ — 出發前核對清單

位於最左側 tab，每次新旅行**獨立**重置，不會跨旅行延續。

### 預設五大分類
| 分類 | 預設項目 |
|---|---|
| 📋 證件 | 護照、簽證、身分證影本、機票確認信 |
| 💳 金融 | 信用卡、現金、外幣換好、行動支付設定 |
| 💊 健康 | 常備藥、旅遊保險、急救包 |
| 🔌 3C | 充電器、萬用轉接頭、行動電源、相機 |
| 🎒 行李 | 換洗衣物、盥洗用品、雨傘、防曬乳 |

### 操作
- **勾選**：點 checkbox 即可，已勾選會顯示刪除線
- **進度條**：頂端顯示完成比例（例如 7 / 19）
- **新增自訂項目**：捲到最下方，選擇分類 → 輸入名稱 → 按 ＋
- **刪除自訂項目**：自訂項目右側會出現 ✕ 按鈕（預設項目無法刪除，只能取消勾選）

## 5. 行程 📅 — 規劃每日行程

主要的行程規劃介面，預設會看到 Day 分頁列、航班卡片區與時間軸。

### 進入編輯模式
右上角 **「✏️ 編輯」** 按鈕，按下後變黃色才能新增 / 刪除 / 拖曳項目。

### 5.1 航班
編輯模式下出現「✈ 新增航班資訊」按鈕，可記錄出發 / 抵達機場、航空、航班號、時間。航班會顯示為登機牌風格卡片。

### 5.2 新增行程
編輯模式下捲到底部，「新增行程」表單：
- **景點 / 活動名稱**：必填
- **類別**：⚔️ 景點 / 🍱 餐廳 / 🛒 購物 / 💤 住宿 / ✈ 機場 / 🎪 體驗
- **第一個行程開始時間**：當天第一個項目才會出現此欄位
- **停留時間**：滑輪選擇小時 + 分鐘

### 5.3 起始時間調整
編輯模式下，每天的時間軸頂端會出現 **「⏰ 當天起始時間」** 輸入框：
- **若已手動設定**：以此為準
- **若未設定且當天有航班**：以最晚一筆航班的抵達時間為起始
- **都沒有**：預設 09:00

> 從第二個行程起的時間皆由「前一個行程開始時間 + 停留時間 + 交通時間」自動推算。

### 5.4 行程項目操作
每個項目卡片含：
- **時間 / 名稱 / 類別 / 停留**
- **📍 Google 地圖**：直接開啟地圖（已標記座標時用座標，否則用名稱搜尋）
- **⠿ 拖曳把手**：長按拖曳調整順序（避免誤觸）
- **編輯按鈕**：修改名稱、類別、停留時間、備註
- **複製到其他天**：可整段行程複製到指定日

### 5.5 交通方式
兩個行程之間會顯示交通段落，可設定：
- **方式**：🚶 步行 / 🚇 地鐵 / 🚌 公車 / 🚕 計程車 / 🚗 自駕 / 🚅 新幹線 / 🚃 電車 / 🔹 其他
- **時間（分）**：影響後續行程時間自動計算

## 6. 地圖 🗺 — 視覺化路線

使用 OpenStreetMap 顯示行程的地理位置。

### 為行程加上座標
1. 在「行程」tab 編輯模式下，點某個項目的 **「📍 設定位置」**
2. 自動跳到地圖頁，銀幕頂端出現提示橫幅
3. 在地圖上點一下要的位置 → 自動回到行程頁

### 地圖頁特色
- **編號標記**：紅色圓圈內顯示該日的順序（1, 2, 3…）
- **彎曲交通虛線**：相鄰兩點間以貝茲曲線連接，顏色依交通方式變化
- **中段標籤**：顯示交通圖示與時間（例：🚇 30 分）

## 7. 記帳 💰 — 花費紀錄

### 介面結構
- **頂端日切換**：選擇某一天查看當日花費
- **當日總額 + 台幣換算**：自動以最新匯率（exchangerate-api.com）換算
- **📊 帳務總覽**：點擊展開全程花費總額、各類別占比、各日花費
- **類別篩選**：可只看某一類別（餐飲、交通…）
- **花費清單**：依時間順序顯示

### 新增 / 編輯花費
編輯模式下右下角會出現 **「＋」** 按鈕：
1. 選擇類別（🍜 餐飲 / 🚌 交通 / 🏨 住宿 / 🎫 票券 / 🛍 購物 / 🧋 小吃 / 💊 其他 / 🚑 緊急）
2. 輸入名稱與金額
3. 選擇日期 → 儲存

點擊已儲存的花費可編輯或刪除。

### 預算追蹤
若建立旅行時有設定預算，總覽會顯示「已用 / 預算（百分比）」，超出時會以紅色警示。

## 8. 美好回憶 ⭐ — 已完成的旅行

從首頁「美好回憶」進入，列出所有封存的旅行。

### 封存方式
- **被動封存**：在進行中旅行又點「冒險開始」時，會詢問是否封存
- **主動封存**：行程頁右上 ⋯ 選單 → 「⭐ 保留回憶」

### 回憶詳情頁
點擊任一回憶卡片進入詳情，可：
- **切換每日**：頂端 D1 / D2 / 星期X 分頁
- **查看完整時間軸**：航班、行程、交通、花費
- **✏️ 複製編輯**：將此回憶複製為目前進行中的行程，可繼續修改後再封存
- **📤 匯出**：Excel/CSV（統一格式）或列印 / PDF

## 9. 匯入 / 匯出 / 備份

### 9.1 統一 CSV 格式（V1.01 起）
匯出的 CSV 採結構化區塊，可在 Excel/Numbers 開啟，**也可由本 App 重新匯入**作為美好回憶。

格式範例：
```
[META]
旅行名稱,東京
出發日期,2026-05-22
回程日期,2026-05-26
貨幣,JPY

[行程]
天數,名稱,類別,備註,停留(分),緯度,經度
1,東京塔,⚔️ 景點,景觀絕佳,120,35.65,139.74
1,淺草寺,⚔️ 景點,,90,,

[交通]
天數,行程順序,交通方式,時間(分),費用
1,1,metro,30,170

[航班]
天數,出發機場,抵達機場,航空,班號,起飛,降落
1,TPE,NRT,長榮,BR198,10:00,14:00

[花費]
天數,類別,名稱,金額
1,🍜,午餐拉麵,1200
```

### 9.2 匯出
- **目前旅行**：行程頁 ⋯ 選單 → 📤 匯出 → 選 Excel/CSV 或 PDF
- **美好回憶**：開啟回憶 → 右上 📤 → Excel/CSV 或 PDF

### 9.3 匯入（存為美好回憶）
1. 進入「美好回憶」頁
2. 點頂端 **「📥 匯入」**
3. 選擇檔案：
   - **`.csv`**（本 App 匯出的統一格式）：保留所有資料含交通與花費
   - **`.xlsx`**（舊版 Excel 行程表格式）：嘗試解析日期欄與行程列

> 雙方都可以用本 App 匯出 CSV 互傳，匯入後即成為對方的美好回憶。

### 9.4 備份 / 還原（完整本機資料）
行程頁 ⋯ 選單：
- **💾 備份資料**：下載 JSON 檔（含所有旅行 + 美好回憶 + 設定）
- **📂 還原備份**：上傳之前的 JSON 檔覆寫本機資料

> 換手機或重灌瀏覽器前建議先備份。

## 10. 常見問題

### Q1：資料會上傳到雲端嗎？
**不會。** 所有資料儲存於瀏覽器 localStorage，僅在本機。匯率查詢會呼叫 exchangerate-api.com 但不送任何個資。

### Q2：清除瀏覽器資料會發生什麼事？
所有旅行、回憶、準備清單會消失。**請定期使用「💾 備份資料」匯出 JSON。**

### Q3：起始時間怎麼計算？
- 若有手動設定（編輯模式下 ⏰）→ 以此為準
- 否則若當天有航班 → 用最晚一筆航班的抵達時間
- 否則 → 預設 09:00

### Q4：刪除美好回憶？
進入回憶列表，每張卡片右上有 🗑 按鈕。

### Q5：可以用一個 CSV 給朋友參考嗎？
可以！匯出 CSV 後傳給對方，對方在自己的 App 裡 美好回憶 → 📥 匯入 即可。

### Q6：版本號在哪？
首頁右下角，目前為 V1.01。
